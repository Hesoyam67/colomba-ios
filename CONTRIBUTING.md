# Contributing

## Branch naming

Use short, scoped branch names:

- `feat/<short-description>` for features
- `bugfix/<short-description>` for bug fixes
- `chore/<short-description>` for maintenance and documentation

## Pull requests

- Use the pull request template.
- Keep PRs focused and reviewable.
- Include a clear test plan.
- Add screenshots or recordings for UI changes.
- Do not merge while CI is red or pending.
- Get at least one reviewer approval before merge.

## Code style

- Run SwiftLint before opening a PR.
- Prefer readable Swift over clever Swift.
- Keep formatting changes separate from behavior changes when possible.
